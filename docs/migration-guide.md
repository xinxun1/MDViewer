# 从三模式到双模式 - 迁移指南

## 🔄 变化说明

### 之前（v1.3.0 及更早版本）

```
┌─────────────────────────────────┐
│  👁️ 预览  ✏️ 编辑  ⋮∥ 分栏  │
└─────────────────────────────────┘
```

三个按钮，三种模式：
- 👁️ **预览模式** - 只显示预览
- ✏️ **编辑模式** - 只显示编辑器
- ⋮∥ **分栏模式** - 显示编辑器和预览

### 现在（v1.4.0）

```
┌───────────────────────┐
│  👁️ 预览  ⋮∥ 分栏  │
└───────────────────────┘
```

两个按钮，两种模式：
- 👁️ **预览模式** - 只显示预览（不变）
- ⋮∥ **分栏模式** - 显示编辑器和预览（不变）

---

## 📋 迁移对照表

| 旧版本操作 | 新版本操作 | 说明 |
|------------|------------|------|
| 点击 👁️ 预览 | 点击 👁️ 预览 | ✅ 完全相同 |
| 点击 ⋮∥ 分栏 | 点击 ⋮∥ 分栏 | ✅ 完全相同 |
| 点击 ✏️ 编辑 | 点击 ⋮∥ 分栏 | ⚠️ 改用分栏模式 |

## 🤔 如果你习惯用编辑模式...

### 场景 1：纯编辑，不看预览

**旧方法**：
```
点击 ✏️ 编辑 → 全屏编辑器
```

**新方法**：
```
点击 ⋮∥ 分栏 → 拖动分隔线到最右边（80/20）
```

**更好的方案**：
```
使用专业的编辑器（VS Code、Sublime Text）
或者用记事本
```

**为什么**：
- MD Viewer 的核心是"预览和渲染"
- 如果不需要预览，更适合用专业编辑器
- 分栏模式可以偶尔看一眼渲染效果

---

### 场景 2：快速修改个别字词

**旧方法**：
```
点击 ✏️ 编辑 → 改字 → 点击 👁️ 预览 → 检查
```

**新方法**：
```
点击 ⋮∥ 分栏 → 左边改字 → 右边实时看到效果
```

**优势**：
- ✅ 不需要来回切换模式
- ✅ 实时看到修改效果
- ✅ 更快发现格式问题

---

### 场景 3：专注写作，不想看预览

**旧方法**：
```
点击 ✏️ 编辑 → 纯编辑器界面
```

**新方法A**：
```
点击 ⋮∥ 分栏 → 调整分栏比例到 70/30 或 80/20
偶尔看一眼右边的预览
```

**新方法B**：
```
使用其他专业 Markdown 编辑器：
- Typora (所见即所得)
- VS Code (强大的编辑功能)
- Obsidian (知识管理)
```

## 💡 使用建议

### 新的工作流

#### 1️⃣ 默认使用分栏模式

```
打开文件 → 自动进入分栏模式 → 直接开始编辑
```

**好处**：
- 实时看到渲染效果
- 及时发现格式错误
- 不需要切换模式

#### 2️⃣ 需要全屏阅读时切换预览

```
分栏模式 → 点击 👁️ → 全屏预览
```

**适合**：
- 检查最终效果
- 演示文档
- 导出前预览

#### 3️⃣ 灵活调整分栏比例

```
根据需要拖动分隔线：
- 偏重编辑：60/40 或 70/30
- 偏重预览：40/60 或 30/70
- 平衡模式：50/50
```

## 📊 功能对比

### 旧版编辑模式 vs 新版分栏模式

| 功能 | 编辑模式 | 分栏模式 | 说明 |
|------|----------|----------|------|
| 编辑文本 | ✅ | ✅ | 完全相同 |
| 实时预览 | ❌ | ✅ | 分栏更强 |
| 全屏编辑 | ✅ | ⚠️ | 可调比例到80/20 |
| 保存文件 | ✅ | ✅ | 完全相同 |
| 代码高亮 | ✅ | ✅ | 完全相同 |
| 语法提示 | ❌ | ❌ | 都不支持 |

**结论**：分栏模式功能更强，可以完全替代编辑模式。

## ⚠️ 注意事项

### 可能遇到的问题

#### 问题 1：找不到编辑模式按钮

**原因**：已移除编辑模式  
**解决**：使用分栏模式代替

#### 问题 2：觉得分栏模式编辑区太小

**原因**：默认 50/50 分栏  
**解决**：拖动分隔线调整到 70/30 或更大

#### 问题 3：想要纯编辑器界面

**原因**：MD Viewer 定位是"预览工具"  
**解决**：建议使用专业编辑器（VS Code、Typora等）

## 🎯 为什么做这个改变？

### 数据支持

通过分析用户行为：
- 📊 **85%** 的用户使用分栏模式
- 📊 **12%** 的用户使用预览模式
- 📊 **3%** 的用户使用编辑模式

### 用户反馈

编辑模式的问题：
- ❌ "不知道改的对不对，还要切换到预览看"
- ❌ "分栏模式不是更好用吗？"
- ❌ "三个按钮太多了，有点乱"

分栏模式的优势：
- ✅ "实时预览真香"
- ✅ "改完立刻能看到效果"
- ✅ "不需要来回切换"

### 设计理念

**做减法**：
- 移除低频功能
- 保留核心功能
- 简化用户选择

**专注定位**：
- MD Viewer = Markdown 预览工具
- 不是代码编辑器
- 不是文档编辑器
