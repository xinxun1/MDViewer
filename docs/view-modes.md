# 视图模式说明

## 当前版本的视图模式

MD Viewer 现在提供**两种视图模式**，满足不同的使用需求：

### 1️⃣ 预览模式 👀

**特点**：
- 📖 纯粹的阅读体验
- 🖥️ 全屏显示预览内容
- 🚫 不显示编辑器
- ⚡ 适合快速浏览文档

**使用场景**：
- 查看文档内容
- 演示 Markdown 渲染效果
- 阅读长篇文章

**快捷键**：点击工具栏的 👁️ 图标

---

### 2️⃣ 分栏模式 ✏️ (默认)

**特点**：
- 📝 左侧编辑器 + 右侧预览
- 🔄 实时同步渲染
- 📏 可调整分栏比例
- 💾 支持保存修改

**使用场景**：
- 编辑 Markdown 文档
- 实时预览渲染效果
- 调整和优化内容格式

**快捷键**：点击工具栏的 ⋮∥ 图标

**分栏调整**：
- 鼠标悬停在中间分隔线
- 拖动调整编辑器和预览区域的大小
- 比例自动保存，下次打开时恢复

---

## 为什么移除编辑模式？

### 之前的三种模式

```
1. 预览模式 - 只看预览
2. 编辑模式 - 只看编辑器  ❌ 已移除
3. 分栏模式 - 编辑 + 预览
```

### 移除原因

**功能重复**：
- 编辑模式 = 只显示编辑器
- 分栏模式 = 编辑器 + 预览
- 如果要编辑，分栏模式更好用（可以实时看到效果）
- 单独的编辑模式实际使用频率很低

**用户反馈**：
- ✅ "我都用分栏模式，可以边改边看"
- ✅ "编辑模式没啥用，还不如记事本"
- ✅ "三个按钮太多了，分不清用哪个"

**简化界面**：
- 减少一个按钮，界面更清爽
- 降低用户学习成本
- 专注于核心功能

## 使用建议

### 💡 最佳实践

**阅读文档时**：
```
使用预览模式 👀
- 全屏阅读，沉浸式体验
- 不会被编辑器分散注意力
```

**编辑文档时**：
```
使用分栏模式 ✏️
- 左边写，右边看
- 实时预览渲染效果
- 及时发现格式问题
```

**调整分栏比例**：
```
根据需求调整：
- 50/50 - 平衡模式（默认）
- 60/40 - 偏重编辑
- 40/60 - 偏重预览
- 20/80 - 主要看预览
- 80/20 - 主要看编辑
```

## 快速切换

### 键盘流

虽然没有提供快捷键，但切换非常方便：

1. **预览** → **分栏**：点击一次 ⋮∥ 按钮
2. **分栏** → **预览**：点击一次 👁️ 按钮

### 工作流程

```
1. 打开文件夹
   ↓
2. 选择文件（自动进入分栏模式）
   ↓
3. 开始编辑和预览
   ↓
4. 需要全屏阅读？切换到预览模式
   ↓
5. 继续编辑？切换回分栏模式
```

## 对比其他编辑器

### Typora 风格
- Typora：即时渲染，编辑即预览
- MD Viewer：分栏模式，编辑和预览分离

### VS Code 风格
- VS Code：分栏模式 + 纯编辑模式
- MD Viewer：分栏模式 + 纯预览模式

**我们的选择**：
- ✅ 保留分栏模式 - 适合编辑和预览
- ✅ 保留预览模式 - 适合纯阅读
- ❌ 移除编辑模式 - 功能被分栏模式覆盖

## 总结

### 两种模式，各司其职

| 模式 | 用途 | 适合场景 |
|------|------|----------|
| 👀 预览模式 | 纯阅读 | 查看文档、演示、学习 |
| ✏️ 分栏模式 | 编辑+预览 | 写作、编辑、调试格式 |

### 设计理念

✅ **简单** - 只保留必要的功能  
✅ **高效** - 分栏模式覆盖90%的编辑需求  
✅ **清爽** - 减少按钮，降低选择困难  
✅ **专注** - 预览模式提供纯粹的阅读体验
